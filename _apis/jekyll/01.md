# Quickstart
<br>
Jekyll은 정적 사이트 생성기(Static site generator)로, 마크업(Markup) 언어로 작성한 텍스트를 레이아웃을 결합하여 정적 웹사이트를 만들어준다. 사이트의 모양과 느낌, URL, 페이지에 표시되는 데이터 등을 원하는 대로 조정할 수 있다.

## Prerequisites

Jekyll을 사용하려면 다음 항목들이 필요하다:

- Ruby 2.7.0 이상 버전
  
- RubyGems
  
- GCC와 Make

자세한 가이드는 [Requirements](#requirements)를 참고하라.

## Instructions

1. 모든 [사전 준비 항목](#installation)을 설치한다.

2. jekyll과 bundler [gems](#gems)을 설치한다.

```terminal
gem install jekyll bundler
```

<ol start="3"><li>./myblog 위치에 새로운 Jekyll 사이트를 생성한다.</li> </ol>

```terminal
jekyll new myblog
```

<ol start="4"><li>생성된 디렉터리로 이동한다.</li> </ol>

```terminal
cd myblog
```

<ol start="5"><li>사이트를 빌드하여 로컬 서버에서 확인한다.</li> </ol>

```terminal
bundle exec jekyll serve
```

<ol start="6"><li>브라우저에서 http://localhost:4000 으로 접속한다.</li> </ol>


<div class="red-caution" markdown="1">
⚠️ Ruby 3.0.0 이상 버전을 사용하는 경우 5번의 <code class="code-inline">bundle exec jekyll serve</code> 실행 시 오류가 발생할 수 있다. 이 경우 <code class="code-inline">webrick</code>을 의존성에 추가한다: <code class="code-inline">bundle add webrick</code>
</div>

<div class="blue-caution" markdown="1">
ℹ️ 자동 새로고침을 원하면 <code class="code-inline">--livereload</code> 옵션을 함께 사용한다:<br>
<code class="code-inline">bundle exec jekyll serve --livereload</code>
</div>


위 과정 진행 중 에러가 발생하면, 먼저 [Requirements](#requirements)에 나온 사전 준비 항목을 모두 설치했는지 확인한다. 지속적으로 문제가 발생할 경우 [Troubleshooting]({% link _apis/jekyll/docs/01/troubleshooting.md %})을 참고하라.

<div class="blue-caution"  markdown="1">
ℹ️ 운영체제에 따라 설치 방식이 달라진다. OS별 설치 [가이드](#guides)를 참고하라.
</div>

# Installation
<br>
Jekyll은 대부분의 시스템에 설치 가능한 [Ruby Gem](#gems) 형태로 배포된다.

## Requirements

- 모든 개발용 헤더 포함 [Ruby](https://www.ruby-lang.org/en/downloads/) 2.7.0 이상 버전 (Ruby 버전 확인: <code class="code-inline">ruby -v</code>)

- [RubyGems](https://rubygems.org/pages/download) (Gems 버전 확인: <code class="code-inline">gem -v</code>)

- [GCC](https://gcc.gnu.org/install/)와 [Make](https://www.gnu.org/software/make/) (버전 확인: <code class="code-inline">gcc -v</code>, <code class="code-inline">g++ -v</code>, <code class="code-inline">make -v</code>)

# Ruby 101
<br>
Jekyll은 Ruby로 작성되었다. Ruby가 처음이라면, 이 페이지에서 주요 용어를 확인할 수 있다.

## Gems
Gem은 Ruby 프로젝트에서 사용할 수 있는 코드 패키지이다. 각 Gem 패키지는 특정 기능들의 묶음으로, 여러 프로젝트 간에 공유하거나 다른 사람들과 나눌 수 있다. 예를 들어, Gem은 다음과 같은 작업을 수행할 수 있다:

- Ruby 객체를 JSON으로 변환

- 페이지네이션(페이지 나누기)

- GitHub 등의 API와 상호작용
  
<br>
Jekyll 자체도 하나의 Gem이다. 많은 Jekyll [플러그인]({% link _apis/jekyll/05.md %})([jekyll-feed](https://github.com/jekyll/jekyll-feed), [jekyll-seo-tag](https://github.com/jekyll/jekyll-seo-tag), [jekyll-archives](https://github.com/jekyll/jekyll-archives) 등) 역시 Gem 형태로 배포된다.

## Gemfile
Gemfile은 사이트에서 사용하는 Gem의 목록 파일이다. 모든 Jekyll 사이트는 메인 폴더(루트 폴더)에 Gemfile을 가지고 있다.

간단한 Jekyll 사이트의 Gemfile 예시는 다음과 같다:

```bash
source "https://rubygems.org"

gem "jekyll"

group :jekyll_plugins do
  gem "jekyll-feed"
  gem "jekyll-seo-tag"
end
```

## Bundler
[Bundler](https://rubygems.org/gems/bundler)는 Gemfile에 적힌 Gem들을 설치해주는 Gem이다.

Gemfile과 Bundler를 반드시 사용해야 하는 것은 아니지만, 다양한 환경에서도 동일한 버전의 Jekyll과 플러그인을 사용할 수 있으므로 적극 권장된다.

Bundler는 <code class="code-inline">gem install bundler</code> 명령어로 설치할 수 있다. 한 번만 설치하면 새로운 Jekyll 프로젝트를 만들 때마다 매번 설치할 필요는 없다.

Gemfile에 적힌 Gem들을 Bundler로 설치하려면, Gemfile이 위치한 디렉터리에서 다음 명령어를 실행한다:

```bash
bundle install
bundle exec jekyll serve
```

Gemfile을 사용하지 않고 Bundler 설치 과정을 생략할 경우, <code class="code-inline">jekyll serve</code> 명령어로 바로 실행 가능하다.

Jekyll에서 Bundler를 사용하는 자세한 방법과 빠르게 실행 환경을 구축하기 위한 가이드는 [Using Jekyll with Bundler]({% link _apis/jekyll/docs/01/using_jekyll_with_bundler.md %}) 문서를 참고하라.

# Community
<br>

## Jekyll Contributor Code of Conduct
이 프로젝트의 기여자이자 유지관리자로서 개방적이고 포용적인 커뮤니티를 만들기 위해, 우리는 이슈 등록, 기능 요청, 문서 업데이트, Pull Request나 패치 제출을 통한 코드 개선 등 다양한 방식으로 기여하는 모든 사람을 존중할 것을 약속합니다.

[Code of conduct]({% link _apis/jekyll/docs/01/code_of_conduct.md %}) 전문 보기

## Reporting Security Vulnerabilities
코드베이스에서 악용될 수 있는 부분을 발견했나요?

사용 버전에 대한 지원이 종료 여부와 문제 보고 절차는 [보안 정책(Security Policy)]({% link _apis/jekyll/docs/01/security_policy.md %})을 참고하세요.

## Where to get support
다음 방법들을 통해 Jekyll과 관련된 도움을 받을 수 있습니다:

- [Jekyll 공식 문서](#quickstart)

- Jekyll을 사용하다가 의문점이 생긴다면 [Jekyll Forum](https://talk.jekyllrb.com/)이나 [StackOverflow](https://stackoverflow.com/questions/tagged/jekyll)에서 문의해보세요.

- [Gitter 채널](https://app.gitter.im/#/room/#jekyll_jekyll:gitter.im)이나 [Libera](irc://irc.libera.chat/#jekyll)의 IRC 채널 #jekyll에 참여하여 다른 Jekyll 사용자들과 의견을 나누세요.

해당 커뮤니티에서 여러 멤버들의 도움을 받아 올바른 방향을 찾을 수 있습니다.

**참고: Jekyll의 Issue Tracker는 지원 포럼이 아닙니다.**
